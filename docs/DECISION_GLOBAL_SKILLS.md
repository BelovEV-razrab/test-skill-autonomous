# Decision Doc: Глобальные skills для Codex (Вариант A)

## Статус
Принято.

## Дата
2026-03-03

## Контекст
Нам нужен повторяемый способ работы со skills в Codex для текущей команды (фаундер + будущий помощник-разработчик). Проект тестовый, без реальных интеграций. Фиксируем только один путь хранения skills: глобально в домашней директории пользователя.

## Решение
Используем **вариант A**: глобальное хранение skills в `~/.codex/skills`.

Для Windows это обычно:
`C:\Users\<username>\.codex\skills`

Для macOS/Linux:
`~/.codex/skills`

## Почему это решение
- Один источник skills для всех локальных проектов на машине.
- Проще онбординг: не нужно дублировать skills по репозиториям.
- Проще сопровождение: ручные обновления делает фаундер централизованно.
- Подходит для тестового и раннего этапа, где важна скорость настройки.

## Ограничения и политика
- Источники skills: только curated и вручную добавленные.
- Обновления skills: только вручную фаундером.
- В проектном репозитории храним документацию и решения в `docs/`, но не копию глобального каталога skills.
- Язык рабочих документов: RU.

## Роли и ответственность
- Фаундер:
  - Добавляет/обновляет skills в `~/.codex/skills`.
  - Валидирует, что skills безопасны и соответствуют policy источников.
  - Обновляет этот Decision Doc при изменении процесса.
- Помощник-разработчик:
  - Использует только утверждённые skills.
  - При проблемах запускает чеклист проверки и фиксирует наблюдения.
  - Не меняет глобальные skills без согласования.

## Процедура настройки (повторяемые шаги)
1. Убедиться, что существует каталог `~/.codex/skills`.
2. Добавить нужный skill из curated-источника или вручную в отдельную папку внутри `skills`.
3. Проверить, что у skill есть `SKILL.md` и корректная структура файлов.
4. Перезапустить/обновить сессию Codex, чтобы перечитать доступные skills.
5. В новом запросе явно вызвать skill по имени и проверить, что он реально применился по инструкции.
6. Если skill не применился, пройти troubleshooting (ниже).

## Definition of Done
Настройка считается завершённой, когда одновременно выполняются оба условия:
- Skill виден в доступных skills текущей сессии.
- Skill реально применяется в ответе/действиях Codex при явном вызове.

## Чеклист проверки
- [ ] Каталог `~/.codex/skills` существует.
- [ ] Нужный skill лежит в отдельной подпапке.
- [ ] В подпапке есть `SKILL.md`.
- [ ] Skill отображается в списке доступных skills в сессии.
- [ ] Skill успешно триггерится по имени.
- [ ] Результат работы соответствует инструкции из `SKILL.md`.
- [ ] Изменения процесса отражены в `docs/`.

## Troubleshooting (минимум)
- Skill не виден:
  - Проверить путь и имя папки в `~/.codex/skills`.
  - Проверить наличие `SKILL.md`.
  - Перезапустить сессию Codex.
- Skill виден, но не применяется:
  - Явно указать точное имя skill в запросе.
  - Проверить, что запрос соответствует назначению skill.
  - Проверить, что `SKILL.md` читаем и не повреждён.

## Пересмотр решения
Решение пересматривается, если:
- Появится требование проектно-изолированных skills.
- Команда вырастет и потребуется более строгий lifecycle/версионирование.
- Изменится политика безопасности источников.
